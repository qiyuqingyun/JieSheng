# Writer's IDE - MVP 目标与下一步计划

## 一、MVP 总体目标

MVP 阶段的核心不是做“功能堆叠”，而是先建立一个稳定、顺手、可扩展的写作工作流。

### 目标 1：统一写作工作台
- 在同一应用中完成章节写作、大纲整理、角色维护
- 避免作者在多个工具间频繁切换

### 目标 2：本地优先与数据可控
- 项目数据以本地文件落盘
- 文件结构清晰、可备份、可迁移

### 目标 3：写作安全
- 自动保存与手动保存并存
- 降低意外关闭、崩溃带来的内容损失

### 目标 4：为后续智能能力预留接口
- 角色卡采用结构化格式（YAML Frontmatter + 富文本正文）
- 章节/大纲/角色之间保留可链接的数据基础

## 二、MVP 交付范围（高层）

### 1. 项目与内容管理
- 新建/打开项目
- 章节与大纲的基础创建、编辑、保存

### 2. 角色卡系统（基础版）
- 角色列表与检索
- 角色卡片化编辑
- 角色关系基础维护
- 头像本地文件选择与项目内存储

### 3. 编辑器与效率能力
- 富文本编辑
- 撤销/重做
- 自动保存（定时 + 失焦）
- 专注模式
- 快捷键

## 三、下一步要做什么

### P1：角色卡体验深化
1. 每个大卡片（基础/静态/动态）内支持自定义小字段
2. 关系选择器升级为更完整的搜索下拉体验
3. 角色卡字段模板（可按题材切换）

### P2：正文联动能力
1. 正文内 `@角色` 提及节点
2. 点击提及弹角色速览
3. 右侧 Pin 多张角色卡伴随写作

### P3：导出与发布能力
1. 导出纯净正文（移除编辑辅助标记）
2. 分章节导出与整书导出
3. 发布前一致性检查（角色名、关系引用）

### P4：版本与协作基础
1. Git 历史查看与关键版本标记
2. 变更对比（章节/大纲/角色）
3. 回滚到指定版本

### P5：AI 前置准备（非直接接入）
1. 结构字段校验与规范化
2. 角色/章节索引准备
3. 可解释的引用链路设计

## 四、完成标准（阶段性）

### MVP 完成标准
- 作者可独立完成：新建项目 -> 写章节 -> 管大纲 -> 管角色 -> 打包发布
- 关键操作无阻塞、无明显数据丢失风险
- 文档与实现保持同步

### 下一阶段进入条件
- 角色卡与正文联动（`@` 提及）可用
- 导出流程稳定
- 版本回滚基础能力可用
